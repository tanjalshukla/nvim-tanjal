## 1. Concept ‚Äî LOCKED

The site **is a Neovim session**. Not "terminal-inspired" ‚Äî a faithful recreation of the chrome: a file-explorer sidebar (the site is a project tree; every page is a file in it), a bufferline showing the buffers you've opened, a content area with a line-number gutter, a lualine-style statusline, and a command-line row. The single signature: **the chrome is functional** ‚Äî vim keys, a working `:` command-line, a Telescope-style finder.

Spend all boldness there. Everything else is disciplined: real content in semantic HTML, dense and bounded (see ¬ß6.0), no gimmicks, fast.

**Audience & the 30-second job:** engineers and recruiters ‚Äî but the site is built to feel like its owner, not to argue a case. In 30 seconds: who he is (photo + about, right on index), what he's building and reading now, and the research links one click away. The craft of the site is the credential; the content is a living snapshot, not a portfolio.

**Forbidden (anti-cringe guardrails):** CRT scanlines/noise, typing animations on real content, fake boot sequences, blinking-cursor spam, autoplaying anything, "I use arch btw" copy, shipping a Nerd Font webfont, ASCII walls as body text. Humor budget: the chrome itself + exactly two easter eggs (`:q`, `:set ascii`). All prose is earnest and professional.

### 5.3 Semantic mapping ‚Äî deploy the palette like a highlighted buffer, not a wireframe

The v1 build failed by rendering nearly everything in greys. Rule: **every screen must visibly use at least three accents**, assigned as One Dark uses them:

- `bg0` buffer ¬∑ `bg-d` chrome panes + desktop behind the window ¬∑ `bg1` cursorline/hover ¬∑ `bg3` `::selection` (with `fg`), borders, winseparators
- `fg` body ¬∑ `light-grey` secondary/inactive ¬∑ `grey` strictly decorative (gutter numbers, `#` heading prefixes, comment-style asides)
- `blue` ‚Äî all links (underlined) and "function"-flavored accents
- `red` ‚Äî headings (bold), per One Dark's markdown convention; also errors/easter eggs
- `green` ‚Äî the statusline NORMAL block (lualine onedark convention), string-flavored accents (the tagline in quotes), git-add-flavored positives
- `purple` ‚Äî VISUAL mode block, keyword-flavored labels (e.g., `last edited:`, list-group titles)
- `yellow` ‚Äî COMMAND mode block, highlights, in-progress markers
- `orange` ‚Äî numbers, dates, progress figures
- `cyan` ‚Äî tags, metadata, tree folder accents

Statusline mode blocks: match lualine's `onedark` theme ‚Äî `NORMAL` green, `VISUAL` purple, `COMMAND` yellow, each with `bg0` text (agent verifies against the lualine theme source).

Type: everything JetBrains Mono. Content column `max-width: 80ch`; body 15‚Äì16px, line-height ‚â• 1.7. Headings differ by color + weight, not exotic sizes: h1/h2 red bold prefixed `# `/`## ` (prefixes in `grey`, decorative `::before`, excluded from copy), h3 orange.

## 6. The chrome ‚Äî LOCKED

### 6.0 The window frame ‚Äî the v2 layout fix

The editor is a **bounded window on a desktop**, not a full-bleed page. Page background: `bg-d` (or 6% darker). The editor window: centered, `max-width: 1080px`, `min-height: 88vh`, `bg0`, 1px `bg3` border, 4px radius, subtle shadow. All chrome (bufferline, explorer, buffer, statusline, cmdline) lives inside it. Density rules: content starts ‚â§ 2 line-heights under the bufferline; no region of the window may be more than ~35% empty at desktop width with real content; explorer entries at comfortable but tight line-height (~1.9). Below ~768px the window goes edge-to-edge (no desktop margin).

### 6.1 File explorer (left pane, primary nav)

`<nav aria-label="explorer">`, nvim-tree-style, rendering the ¬ß3 tree, always expanded. Box-drawing glyphs (`‚îú‚îÄ‚îÄ`, `‚îî‚îÄ‚îÄ`, `‚îÇ`) are decorative `::before` content, `aria-hidden`, unselectable, **with a space before filenames**. Pane: `bg-d`, ~220px, 1px `bg3` winseparator on its right. Current file: `fg` on `bg1` cursorline row; others `light-grey`; hover `bg1`. Root `~/tanjal` in `cyan`. Below desktop: hidden by default, opens as an overlay drawer from the statusline's `‚â° files` button (and `Space e`); drawer must work without the island (`<details>`-based acceptable if keyboard- and screen-reader-clean). Plain anchor links ‚Äî full navigation works with JS disabled.

### 6.2 Bufferline (top, secondary nav, labelled)

Shows **buffers opened this session**, like a real bufferline. Static default: current file only. The island (¬ß7) appends visited files (sessionStorage): active tab `fg` on `bg0` with a `green` `‚óè`; inactive `light-grey` on `bg-d`. `]b`/`[b` cycle open buffers. Horizontal scroll, no wrap.

### 6.3 Buffer area

`Buffer.astro`: line-number gutter left (CSS counters), content right, sits beside the explorer. Gutter numbers `grey`, `aria-hidden`, `user-select: none`, and cover **every block-level element** (paragraphs, headings, list items) so numbering reads continuous rather than floating. Gutter hides below ~480px. **Semantics test:** with CSS removed, every page reads as a plain, sensible document.

### 6.4 Statusline (bottom of window, `<footer>`, sticky within it)

Lualine layout, left ‚Üí right: mode block (¬ß5.3 colors) ¬∑ ` main` (decorative) ¬∑ `~/tanjal/<file>` ¬∑ _(spacer)_ ¬∑ contact links `gh` `in` `@` ¬∑ theme toggle (`‚òÄ/‚òæ`, equivalent to `:set bg=`) ¬∑ `‚â° files` (below desktop) ¬∑ scroll `%` + `Ln:Col` (island-updated; static `Top 0:0` without it). Nothing else ‚Äî remove any unspecced icons. Mode is honest: `NORMAL` default, `VISUAL` on text selection, `COMMAND` while the command-line has focus.

### 6.5 Command-line row

A `:`-prompt row at the window's bottom edge. Click/tap `:` or press `:` to focus ‚Äî a real `<input>` so mobile keyboards work.

## 7. Keyboard & command layer ‚Äî the one island ‚Äî LOCKED behavior

Progressive enhancement; every destination reachable by plain links. Island budget: **< 15KB gzipped**.

Keys (inactive when focus is in an input; never bind Ctrl/Cmd combos):

- `j`/`k` scroll ¬∑ `gg`/`G` top/bottom (instant under `prefers-reduced-motion`)
- `]b`/`[b` cycle open buffers (falls back to all files when only one is open)
- `Space e` toggle the explorer (drawer below desktop)
- `:` command-line ¬∑ `/` or `Space f f` fuzzy finder ¬∑ `?` help overlay ¬∑ `Esc` closes any overlay, restores focus

Command grammar:

- `:e <file>` navigate (Tab-completes) ¬∑ `:e!` hard reload
- `:help` keymap overlay
- `:set bg=light|dark` theme (persisted, no-flash inline script)
- `:set ascii` / `:set noascii` ‚Äî index photo ‚Üî pre-generated ASCII art, persisted
- `:q` ‚Üí statusline flashes `E37: No write since last change` in red; `:q!` ‚Üí brief "goodbye." toast, scroll to top
- Unknown ‚Üí `E492: Not an editor command: <input>` in red. All error strings are authentic Vim errors.

Fuzzy finder (Telescope-style): `bg-d` modal over the window, `> ` prompt, subsequence match with matched chars in `yellow`; entries = files + top-level sections + external links (GitHub, LinkedIn, Hedwig landing page, paper). Arrows + Enter. Focus trapped while open, returned on close; labelled `role="dialog"` + `aria-modal`.

## 9. Accessibility ‚Äî LOCKED

- Touch: every key-driven action has a visible tap path (explorer links, `‚â° files`, `:` button, finder button).
